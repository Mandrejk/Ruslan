Пример использования Framework

Для использования Framework необходимо сделать следующее:

  • определить функции FrameworkInit() и FrameworkDeInit();
  • в функции FrameworkInit() создать объект менеджера потоков следующим образом: Framework::gpThreadManager = new Framework::classThreadManager(argc,argv);

Точками входа для инициализации и деинициализации Framework являются функции FrameworkInit() и FrameworkDeInit(). Кроме этого возможно создать класс наследник менеджера потоков и определить его методы Initialize() и DeInitialize(), и в этих методах осуществлять соответсвенно инициализацию и деинициализацию приложения.

Пример - класс HelloThreadManager из HelloThread.h.

Все потоки должны быть реализованы как наследники класса classThreadInstance.

Наследники могут реализовывать методы:

  • virtual void ProcessMessage(MessageEntryPtr message);
  • virtual void ProcessFdEvent(int fd, unsigned int events);
  • virtual void ProcessTimerEvent(unsigned int timerId);
  • virtual void PreWorkInit(void* pData);
  • virtual void PostWorkInit(void* pData);

Перед работой потока будет вызван метод PreWorkInit() класса потока, после работы будет вызван метод PostWorkInit().

Сообщение создается следующим образом:

Framework::ThreadMessagePtr message = Framework::classThreadMessage::CreateThreadMessage(thread->GetId(),0,0);

Объекты создаются и добавляются в сообщение следующим образом:

Framework::typeMessageObjectPtr objPtrThreadId(new Framework::typeThreadId(threadId_)); sendMessage->AddObject(0,objPtrThreadId);

Объекты получаются из сообщения следующим образом:

Framework::typeThreadId* threadId = static_cast<Framework::typeThreadId*>(message->GetObject(0));

Потоки должны быть созданы с помощью макроса ThreadManager_CreateThread(classThreadInstance* pThread, void* pData)
