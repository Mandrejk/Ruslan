Detailed Description

Класс потока

Все потоки должны быть реализованы как наследники класса classThreadInstance. classThreadInstance обечивает взаимодействие с потоковым менеджером:

  • регистрируется при создании, получая идентификационный номер (id);
  • ThreadManager в соответствии с id отправляет сообщения потоку;
  • отправляет менеджеру сообщеня о своей живучести.

Наследники могут реализовывать методы:

  • virtual void ProcessMessage(MessageEntryPtr message);
  • virtual void ProcessFdEvent(int fd, unsigned int events);
  • virtual void ProcessTimerEvent(unsigned int timerId);
  • virtual void PreWorkInit(void* pData);
  • virtual void PostWorkInit(void* pData);

Обработка сообщений и событий (ProcessMessage, ProcessFdEvent и ProcessTimerEvent) не должна превышать по времени заданного таймаута, иначе менеджер потоков зафиксирует ошибку. Таймаут задается функцией SetLiveTimeout.

Потоки должны быть созданы с помощью макроса ThreadManager_CreateThread(). Макрос запишет имя файла, номер строки и имя функции из которой был вызван поток в имя потока. Так же макрос запустит поток.

Пример использования classThreadInstance:

class NewThread : public classThreadInstance { NewThread(void* pData) : classThreadInstance(pData) { ... } virtual ~NewThread;

virtual void ProcessMessage(MessageEntryPtr message); virtual void ProcessFdEvent(int fd, unsigned int events); virtual void PreWorkInit(void* pData); virtual void PostWorkInit(void* pData); }

NewThread* thread = static_cast<NewThread*>(ThreadManager_CreateThread(NewThread,0));
